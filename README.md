## Project Overview
This repository contains a professional Data Analyst portfolio project focused on credit card customer and transaction analytics. The dashboard is built in **Power BI** using **SQL** transformations and **CSV** datasets to provide actionable business insights.

## Project Objectives
- Track financial KPIs for credit card operations.
- Understand customer behavior by segment, age, education, and card category.
- Identify revenue-driving spending patterns for business decision-making.
- Present insights in an executive-friendly and interactive dashboard format.

## Features
- Interactive Power BI dashboard with slicers and drill-down views.
- KPI tracking for revenue, income, interest earned, and transaction metrics.
- Customer segmentation analysis for targeted strategy.
- Expenditure pattern analysis across demographics and card categories.

## Dashboard KPIs & Analysis Areas
- Revenue
- Interest Earned
- Income
- Transaction Amount
- Transaction Count
- Customer Segmentation
- Age Group Analysis
- Education Analysis
- Card Category Analysis
- Expenditure Analysis

## Key Business Insights
- High-value customer segments contribute disproportionately to overall revenue.
- Certain age groups and education segments show stronger transaction frequency.
- Card category adoption impacts both transaction count and interest generation.
- Spending category trends reveal opportunities for targeted offers and retention campaigns.

- `dashboard/screenshots/`: Dashboard preview images for this README.
- `data/raw/`: Source CSV files.
- `data/processed/`: Cleaned or transformed data used for analysis.
- `sql/`: SQL scripts used for data extraction and transformation.
- `docs/`: Supporting project documentation and business notes.

## Installation & Usage
1. Clone the repository:
   ```bash
   git clone https://github.com/Mohitbhardwaj2403/Credit-Card-Financial-Dashboard-.git
   ```
   > Note: The repository name intentionally ends with a trailing hyphen (`-`).
2. Place source files in `data/raw/`.
3. Run SQL scripts from the `sql/` folder to prepare analysis-ready datasets.
4. Open the Power BI `.pbix` file from `dashboard/`.
5. Refresh data connections and explore insights interactively.
